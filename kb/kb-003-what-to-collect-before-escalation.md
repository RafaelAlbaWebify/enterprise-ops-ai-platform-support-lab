# KB-003 — What to Collect Before Escalation

## Purpose

Make escalations clear, useful and actionable.

## Minimum information

- User or application affected
- Team or project
- Business impact
- Timestamp and timezone
- Exact error message
- Screenshot if useful
- Request ID or correlation ID
- Endpoint or service URL
- Steps to reproduce
- Number of affected users
- Recent changes
- Troubleshooting already performed

## Good escalation example

> User A from Project X receives 403 when calling endpoint Y since 09:20 CET. Other users in the same project can access it. Group membership was checked and appears different from working users. Request ID attached. Please review RBAC/group assignment.

## Poor escalation example

> AI is not working. Please check.
