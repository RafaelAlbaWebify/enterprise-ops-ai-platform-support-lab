# KB-005 — Basic Monitoring for Operations

## Purpose

Monitoring helps detect service problems before or during user impact.

## Basic monitoring areas

- Availability
- Latency
- Error rate
- Resource usage
- Authentication failures
- Failed deployments
- Quota usage
- Service health

## Logs vs metrics vs alerts

| Area | Meaning |
|---|---|
| Logs | Detailed event records |
| Metrics | Numeric measurements over time |
| Alerts | Notifications when thresholds or conditions are met |

## Support value

Good monitoring helps support teams understand:

- What happened
- When it happened
- Who was affected
- Whether the issue is still active
- Whether escalation is required
